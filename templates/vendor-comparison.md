## 템플릿 사용 가이드

<!-- 이 섹션은 실제 문서에서 삭제 -->

### 작성 순서

1. **PRD-001 읽기**: 전체 규칙을 먼저 숙지
2. **[대괄호] 채우기**: 빈 칸을 채우면서 구조 완성
3. **HTML 주석 삭제**: 가이드 주석 전체 제거
4. **DO/DO NOT 체크리스트 검증**: PRD-001 체크리스트로 최종 검수

### 주요 작성 원칙

| 원칙 | 내용 |
|------|------|
| **5단계 구조** | 과정 요약 - 핵심 비교 - 중간 정리 |
| **인상-반전-심층** | 표면 인상 → "하지만" → 심층 분석 → "결국" → 수렴 |
| **종결어미 3단계** | 사실(~습니다), 추정(~보입니다), 판단(~판단됩니다) |
| **비유 1회** | 도입부에 한국 독자 친숙한 비유 1회만 |
| **팩트 70% : 의견 30%** | 의견은 `**의견**` 헤더로 분리 |

### 절대 금지 사항

- 금지: `한글(영문)` 병기 (예: "대역폭(Bandwidth)" → "Bandwidth" 또는 "대역폭" 중 하나만)
- 금지: 이모지/아이콘
- 금지: 1인칭 ("저는", "제가")
- 금지: 문어체 접속사 ("그러나", "그렇지만")
- 금지: "결론", "최종 추천" 헤더
- 금지: Pros/Cons 테이블
- 금지: 페이지 하단 각주 (인라인 하이퍼링크는 허용)
- 금지: 목차 (A4 3매 이하 문서)
- 금지: "다음으로", "이어서" 전환 문구
- 금지: 한 문장에 볼드 2회 이상
- 금지: 순서 없는 항목에 번호 매기기 (핵심 대비 2항목은 번호 허용)
