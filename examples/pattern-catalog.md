```
L115 (주장): Vimeo 의 경우 Templete 수준의 변경만 가능합니다.
L117-130 (기능 나열 불릿)
L132-134 (증거): File ID 2개 (스크린샷)

```

주장 → 증거 → 해석 순서. 이미지를 단독으로 놓지 않음.

디자인 도구 선정 보고서에서:

**주장**: Figma는 컴포넌트 변형(Variant) 기능으로 디자인 시스템을 효율적으로 관리할 수 있다.

**증거**: (Figma 컴포넌트 패널 스크린샷)

**해석**: 버튼 하나로 10개 스타일을 관리할 수 있어 **디자인 일관성 유지가 자동화된다.**

### N20: 번호 vs 일반 불릿 구분

**Canvas 근거**: L21 (data-section-style='6') vs L117 (data-section-style='5')
**PRD v2.0**: 섹션 9 (수정)

**원문 인용**:

```
L21 (번호 불릿, style='6'):
1. Brightcove: 비싸고 자유도 높음.
2. Vimeo: 싸고 자유도 낮음.

L117 (일반 불릿, style='5'):
- 로고
- 스플래시 화면
- 컬러 테마
...
```

핵심 대비 2항목은 번호 불릿, 기능 나열은 일반 불릿.

**범용 적용 예시**:

채팅 솔루션 선정 보고서에서:

**핵심 대비** (번호 불릿):
1. SendBird: 비싸고 커스터마이징 자유도 높음.
2. Stream Chat: 저렴하고 표준 UI만 제공.

**기능 나열** (일반 불릿):
- 파일 첨부
- 읽음 표시
- 푸시 알림
- 멘션 기능

순서가 중요하면 번호, 순서가 무의미하면 일반 불릿.
