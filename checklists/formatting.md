# 포맷 검증 체크리스트

PRD-001 v2.0 섹션 6, 9 기반 포맷팅 검증.

---

## 자동 검증 항목

### 볼드 (PRD 섹션 6.3)

- [ ] **[AUTO-11]** 한 문장 내 볼드가 2회 이상 없는가
  - 검증 방법: 정규식 - 같은 문장 내 `**...**` 카운트
  - 기준: 1회만 허용

### 금액 (PRD 섹션 6.4)

- [ ] **[AUTO-10]** 금액 형식이 올바른가
  - 검증 방법: 정규식 `\$\s?\d{1,3}(,\d{3})*`
  - 기준: `$ 금액` (공백 1칸, 한화 금지)
  - 예시: `$ 750,000` ✅ / `$750,000` ❌

### 테이블 (PRD 섹션 6.2)

- [ ] **[AUTO-15]** 테이블이 3행x3열 이하인가
  - 검증 방법: Markdown 테이블 행/열 카운트
  - 기준: 최대 3행x3열

### 불릿 (PRD 섹션 6.1)

- [ ] **[AUTO-16]** 불릿 내에 합쇼체가 없는가
  - 검증 방법: 정규식 - 불릿 줄 내 `습니다|됩니다|입니다` 검출
  - 기준: 불릿은 명사형 또는 해체 종결
  - 예시:
    - ✅ `로고`, `스플래시 화면` (명사형)
    - ✅ `템플릿 외에 추가할 계획이 없다.` (해체)
    - ❌ `로고를 추가합니다` (합쇼체)

### 금지 표현 (PRD 섹션 6.5, 9)

- [ ] **[AUTO-7]** 한글(영문) 병기가 없는가
  - 검증 방법: 정규식 `[가-힣]+\([A-Za-z]+\)`
  - 기준: "시디엔(CDN)" 형식 금지

- [ ] **[AUTO-8]** 이모지/아이콘이 없는가
  - 검증 방법: Unicode emoji range 검출
  - 기준: 전체 문서 내 이모지 0개

- [ ] **[AUTO-9]** "결론", "최종 추천" 헤더가 없는가
  - 검증 방법: 정규식 `^#{1,3}\s*(결론|최종 추천)`
  - 기준: 해당 헤더 사용 금지

- [ ] **[AUTO-14]** "감사합니다" 인사말이 없는가
  - 검증 방법: 정규식 `감사합니다`
  - 기준: 전체 문서 내 0회

- [ ] **[AUTO-17]** 목차/ToC가 없는가
  - 검증 방법: 정규식 `목차|Table of Contents`
  - 기준: A4 3매 이하 문서에 목차 불필요

- [ ] **[AUTO-19]** Pros/Cons 테이블이 없는가
  - 검증 방법: 정규식 `Pros|Cons|장점|단점` (테이블 헤더 내)
  - 기준: 서술형으로 대체

- [ ] **[AUTO-20]** 점수/별점/등급이 없는가
  - 검증 방법: 정규식 `[0-9]+점|★|⭐|[0-9]+\/[0-9]+`
  - 기준: 정량적 평가 표시 금지

## 수동 검증 항목

### 볼드 스캔 (PRD 섹션 6.3)

- [ ] **[MANUAL-7]** 볼드만 읽어도 핵심이 파악되는가
  - 판단 기준: 볼드 항목만 추출해 읽기 테스트
  - 확인 사항:
    - 스캔 포인트: 업체명, 금액, 선택지 헤더, 의견 헤더, 핵심 판단
    - 볼드만으로 문서 흐름 이해 가능

### 이미지 배치 (PRD 섹션 6.6)

- [ ] **[MANUAL-11]** 이미지가 "주장-증거-해석" 3단 구조인가
  - 판단 기준: 이미지 전후 텍스트의 역할 일치
  - 확인 사항:
    - 이미지 전: 주장 텍스트
    - 이미지: 시각적 증거
    - 이미지 후: 해석 텍스트
    - 이미지를 단독으로 놓지 않음

### 나열-부정 클로징 (PRD 섹션 6.1)

- [ ] **[MANUAL-15]** 나열-부정 클로징이 적절한가
  - 판단 기준: 마지막 불릿의 부정 종결 적절성
  - 확인 사항:
    - 긍정 나열 6-7개
    - 마지막 1개는 "~안됨"으로 한계 명시

### Draft-Polish 완성도 (PRD 섹션 12)

- [ ] **[MANUAL-16]** Draft-Polish 워크플로우가 완료되었는가
  - 판단 기준: 볼드, 과금유형, 조건 상세화 적용
  - 확인 사항:
    - Draft 체크포인트: 구조, 핵심 팩트, 개략 금액
    - Polish 완료: 볼드 추가, 과금 유형 병기, 조건 상세화, H2 세분화, 이미지 삽입
